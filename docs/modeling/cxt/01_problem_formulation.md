# CxT — Problem Formulation

## 1. Background

Expected Goals (xG) and its contextual extension CxG measure the value of a
**shot**. Contextual Action Value (CxA) measures the expected goal threat
generated by **individual creative actions** — passes, carries, crosses, and
cutbacks — within a possession. Both metrics assign value to discrete events.

Neither answers a more fundamental question:

> *How much did the game-state improve because of this action?*

**Contextual Threat Value (CxT)** answers this question by measuring the
**change in the possessing team's expected goal threat** caused by each action.
It is grounded in the state-value framework from reinforcement learning: every
possession is a sequence of states, and each action transitions the game from
one state to another. CxT quantifies the value added by that transition.

This formulation has several advantages over shot-based metrics:

- It rewards actions that improve position even when no shot follows.
- It can penalise actions that lose dangerous possession states (negative values).
- It provides a unified currency — expected goal threat — across all action types.
- It is not dependent on whether a shot eventually occurs in the possession.

---

## 2. Formal Problem Statement

### 2.1 State-Value Function

Let $s$ denote a **game state** — a vector of observable features describing
the location, context, and situation of the possession at a given moment.

Define the **state-value function** $V(s)$ as the expected discounted CxG the
attacking team will generate from possession state $s$:

$$V(s) = \mathbb{E}\!\left[\sum_{k=0}^{K} \gamma^k \cdot \text{CxG}_k \;\middle|\; S = s\right]$$

where:
- $K$ is the number of shots in the remainder of the possession
- $\text{CxG}_k$ is the CxG value of the $k$-th subsequent shot
- $\gamma = 0.9$ is the discount factor (shots earlier in the possession are
  weighted more highly, reflecting that the threat is more directly caused by
  the current action)

### 2.2 CxT as a State-Value Difference

Given action $i$ that transitions the game from state $s_{\text{before}}$ to
state $s_{\text{after}}$, the CxT score is:

$$\boxed{\text{CxT}_i = V(s_{\text{after},i}) - V(s_{\text{before},i})}$$

A positive CxT indicates that the action moved the team into a higher-threat
state — more likely to generate dangerous shooting opportunities. A negative
CxT indicates that the action deteriorated the possession state (e.g. a
misplaced pass into a crowded area).

This is structurally analogous to the TD-difference (temporal difference) used
to train value functions in reinforcement learning.

### 2.3 Training Target

The state-value function $V(s)$ is estimated by a regression model. The
training target for each action row is:

$$y_i = \underbrace{\sum_{k=0}^{K_i} \gamma^k \cdot \text{CxG}_{i,k}}_{\texttt{possession\_cxg}}$$

where $K_i$ is the number of shots in the same possession as action $i$ and
$\text{CxG}_{i,k}$ is the CxG of the $k$-th shot (0-indexed).

All actions within the same possession share the same `possession_cxg` value.
This means the model learns: *given this game state, what is the expected total
(discounted) CxG threat that will emerge from this possession?*

The CxT score is then computed by scoring each action twice — once with
before-state features, once with after-state features (via column remapping) —
and taking the difference.

---

## 3. Target Distribution

The `possession_cxg` target has a heavily **zero-inflated** distribution:

| Statistic | Value |
|---|---|
| Total training actions | 220,899 |
| Actions with `possession_cxg = 0` | ~84% |
| Actions with `possession_cxg > 0` | ~16% |
| Mean (all rows) | ~0.013 |
| Mean (positive rows only) | ~0.08 |
| 95th percentile (positive) | ~0.30 |

84% of creative actions occur in possessions that produce no shot. This is the
defining statistical challenge of CxT modelling: most targets are exactly zero
(discrete mass), while the positive tail has a roughly Gamma-like distribution
(skewed, bounded at zero, no upper limit).

## 4. Relationship to CxG and CxA

CxG, CxA, and CxT form a complementary three-layer framework:

| Metric | Unit of analysis | Question answered |
|---|---|---|
| CxG | Shot | How likely is this shot to result in a goal? |
| CxA | Creative action | How much expected goal threat does this action generate (per possession)? |
| CxT | Creative action | How much did this action improve the possession state? |

CxT differs from CxA in two key respects:

1. **CxA** conditions on whether a shot follows in the possession; **CxT** does
   not — it assigns value regardless of whether the current possession produces
   a shot.
2. **CxA** is a single-stage prediction; **CxT** is a difference of two state
   evaluations.

In practice, CxT provides finer granularity for actions that improve position
without directly creating shots, such as progressive carries in midfield or
switches of play that open wide channels.

CxT depends on CxG: the `possession_cxg` target is constructed by scoring each
shot in the training data with the production CxG model
(`models/cxg/glm_contextual.joblib`), then computing the discounted sum per
possession. CxT is therefore a downstream consumer of CxG predictions.

---

## 5. Key Modelling Constraints

**C1 — No target leakage.** The `possession_cxg` value for an action must be
computed from shots in the same possession — not from future possessions or
future matches. The discount is applied by shot index within the possession.

**C2 — Inline CxG scoring.** `possession_cxg` is not stored in the features
parquet. It is computed at training time by loading the production CxG model
and scoring shot rows inline (`scripts/train_cxt.py::_attach_cxg_scores`).
This ensures that if the CxG model is updated, the CxT target updates
consistently.

**C3 — Before/after state remapping.** The state-value model $V(s)$ is trained
on before-state features. To compute $V(s_{\text{after}})$, the after-state
columns (`end_x`, `end_y`, `end_distance_to_goal`, etc.) are remapped to their
before-state equivalents before calling `predict()`. This reuse of a single
model for both states is enforced by the `BEFORE_TO_AFTER` mapping in
`src/models/cxt/feature_sets.py`.

**C4 — Match-level split integrity.** CV and train/test splits must keep all
actions from a single match together. Context features (opponent ratings, match
state) are correlated within a match; row-level splitting would inflate CV
scores by allowing the model to see near-identical context vectors in both
train and validation.

## 6. Dataset

| Split | Competition | Season | Matches | Creative actions | Positive rate |
|---|---|---|---|---|---|
| Train/Val (CV) | FIFA World Cup | 2022 | 64 | ~135,000 | ~16% |
| Train (CV) | UEFA Euro | 2020 | 51 | ~85,000 | ~16% |
| **CV pool total** | — | — | **115** | **220,899** | **~16%** |
| **Held-out test** | UEFA Euro | 2024 | 51 | **98,029** | ~19.6% |

Creative actions include: `pass`, `carry`, `cross`, `cutback`. Shots and
other event types are excluded from the CxT training set (shots are scored
by CxG; other events do not have meaningful before/after state vectors).

The held-out test set (Euro 2024) was **never seen during any training fold**
and was not used for model selection.
